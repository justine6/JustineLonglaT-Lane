# Automating Infrastructure with Terraform

## Patterns
- **Workspaces per env** (dev/stage/prod).
- **Modules** for DRY resources (VPC, ECS/EKS, RDS).
- **Pipelines** with plan/apply gates and OPA policy checks.

## State & Security
- Remote state in S3 + DynamoDB lock; least-privilege IAM roles.
